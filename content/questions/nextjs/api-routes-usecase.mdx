# Deep Answer

### Interview Answer (Aise bolna hai):
API routes main tab use karta hoon jab mujhe frontend aur backend ke beech ek lightweight “middle layer” chahiye ho. Especially jab secrets hide karne hain ya third-party API call server side se karni hai.

Example: frontend directly payment provider ya secure token exchange ko call na kare. Main API route bana deta hoon jo server side se call kare and safe response de.

But main API routes ko full backend replacement ki tarah treat nahi karta. Agar system large hai, to dedicated backend services best hoti hain. API routes = integration glue and lightweight utilities, that’s it.
