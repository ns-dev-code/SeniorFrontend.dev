# Deep Answer

### Interview Answer (Aise bolna hai):
Hydration mismatch basically tab hota hai jab server ne jo HTML bheja, client usko hydrate karte time different output produce kar deta hai. User ko kabhi-kabhi flicker dikhta hai, console me hydration warnings aati hain, aur kabhi UI break bhi ho jaata hai.

Ye issue usually tab aata hai jab server render aur client render me non-deterministic cheezein hoti hain â€” jaise random IDs, current time, window size based rendering, ya locale differences.

Main isko fix karne ke liye simple discipline follow karta hoon: server render should be deterministic. Agar koi cheez browser-only hai, like window ya localStorage, main usko useEffect ke andar shift karta hoon, ya client-only dynamic import karta hoon. Aisa karne se SSR clean rehta hai aur hydration stable ho jaati hai.
