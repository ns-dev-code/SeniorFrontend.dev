# Deep Answer

### Interview Answer (Aise bolna hai):
BFF pattern ka main idea ye hota hai ki frontend backend se directly baat na kare, balki ek thin server layer ho jo frontend ke needs ke hisaab se data shape kare.

Next.js API routes iske liye kaafi achha fit hain. Maine unko use kiya hai jab mujhe multiple backend APIs ko combine karna hota hai ya sensitive logic hide karna hota hai. Example ke liye, frontend ke liye ek simple response chahiye hota hai, par backend side pe 3 APIs call ho rahi hoti hain.

BFF se frontend code simple rehta hai aur backend contracts loosely coupled ho jaate hain. Par main ye bhi dhyaan rakhta hoon ki API routes ko full backend banane ki koshish na ho.
