# Deep Answer

### Interview Answer (Aise bolna hai):
Naye developer ke liye sabse badi problem hoti hai — “main start kahan se karun?” Isliye architecture clarity bahut important hoti hai.

Main hamesha feature boundaries clear rakhta hoon taaki new joiner ek feature ko samajh ke confident ho jaaye. Main unko pehle shared components ya core flows explain karta hoon. Documentation light rakhta hoon, par enough hoti hai taaki confusion na ho. Pair programming aur code reviews se unko context milta hai.
